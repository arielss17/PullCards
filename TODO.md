1. Se o usuario acessar qualquer pagina sem estar logado, deve ser redirecionado para a tela de login sem exibir nenhum dado da pagina que ele tentou acessar deslogado.
2. Com o usuario logado, ao cliar no próprio nome, deve ser redirecionado para a tela de configuração do usuario permitindo alterar nome e senha.
3. Devemos adicionar uma classificação ao arquivo "monsters.json" como se ele fosse uma "expansão" do jogo, permitindo que futuramente possamos adicionar novas coleções de cartas sem ter que modificar o código, para que elas sejam consideradas de outra coleção. Para realizar essa alteração, precisamos criar uma página para gerenciar as expansões na área Administrativa, que permita o administrador selecionar o arquivo JSON respectivo e "batizar" a expansão. Dessa forma estaremos aptos a criar novas expansões, selecionando outros arquivos JSON que tenham a mesma estrutura. Também se faz importante adicionar um ID único nas cartas e um "id da carta na expansão", que vai de 1 a N para permitir identificar as cartas de uma expansão. Cada expansão deverá ter sua configuração de TIERS e MESAS (tables) que serão usadas para calcular a probabilidade de summonar uma carta.
4. Após a implementação do ítem 3, devemos implementar uma tela de seleção de expansão para o jogador, que permita ao usuario selecionar a expansão que ele deseja summonar sua carta. 
5. Após implementação do ítem 4, devemos ajustar a tela de coleção para permitir a seleção de uma expansão, que irá filtrar as cartas que o jogador possui e que ele não possui, permitindo que ele veja as cartas que ele possui de cada coleção.
6. Ainda na visualização da coleção, se o jogador não possuir a carta, ao invés de exibir a carta em preto e branco, devemos exibir um Frame com o "Id da carta na expansão", sem revelar qual carta ele não tem, pois só conseguirá saber após conseguir summonar a carta.
7. Devemos permitir o Admin selecionar qual é a "Expansão Destaque", que será expansão pré-selecionada na tela de summon e na tela de coleção do jogador.
8. Por fim, devemos permitir que o jogador selecione qual expansão ele deseja utilizar uma carga para summonar uma carta.
9. É preciso permitir que o Admin selecione as cores de destaque de cada Table e Tier, hoje os efeitos estão com cores "hardcoded", precisa permitir que tanto o efeito após roll do d100 da table quanto o efeito da cor na borda da carta seja personalizável de acordo com a table, expansão e tier da carta.
